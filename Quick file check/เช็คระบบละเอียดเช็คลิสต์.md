1. Check 09: awk ไม่เจอ DEPENDENCIES section (header ใช้ `* DEPENDENCIES:` แต่ awk อาจจะไม่ match)
2. Check 10: มี unbound variable error + ไม่เจอ ARCHITECTURE section เหมือนกัน

ขอตรวจ awk pattern ใหม่
awk pattern ไม่ match เพราะบรรทัดจริงขึ้นต้นด้วย ` * DEPENDENCIES:` (space + asterisk + space) ไม่ใช่ `* DEPENDENCIES:` ขอตรวจให้ละเอียด
เห็นแล้ว — บรรทัดเริ่มต้นด้วย ` * ` (1 space + asterisk + 1 space) แต่ awk regex `/^\s*\*\s*DEPENDENCIES:/` ใช้ `\s` ซึ่ง awk มาตรฐาน (mawk/gawk default) ไม่รองรับ — ต้องใช้ `[ \t]` แทน ขอแก้
ตอนนี้แก้ unbound variable error ใน check 10 — `declare -A unresolved_funcs` + `set -u` ทำงานไม่ถูกต้องเมื่อ access `${unresolved_funcs[$func]}` บน key ที่ไม่มีอยู่

- **ห้ามทำตอนนี้** — คุณยังไม่ได้ยิงข้อมูลจริง จะ optimize ผิดที่
- **ทำตาม priority ที่ผมเรียงให้** (Phase 3 → 5.2 → 7 → ดู baseline → เลือกต่อ)
- **ข้าม Phase 4.2 กับ 2.3** ถ้าไม่จำเป็นจริงๆ (เสี่ยง quota)

### สิ่งที่ผมแนะนำให้ทำอาทิตย์นี้
1. แก้ 3 Critical Issues ใน PR เดียว (30 นาที)
2. Deploy ขึ้น staging + ยิงข้อมูลจริง 10-20 แถว → ดูผล
3. ยิงข้อมูลจริง full scale → รอ 2-4 สัปดาห์ → capture baseline
4. ดู baseline แล้วค่อยเลือก Phase แรกของ V6.0 ที่จะทำ

อยากให้ผมช่วยแก้ 3 Critical Issues ให้เลยไหมครับ เพื่อให้คุณไปยิงข้อมูลจริงได้เลย? หรืออยากคุยเรื่อง priority ของ V6.0 phases เพิ่มเติม?
